    Protección de los addons

    <> = requerido, [] = opcional, ; = anotación
    Tus addons pueden ser codificados para que la fuente sea ilegible / inalterable. Además, se les puede agregar una whitelist/blacklist para que solo ciertos UUID's puedan ejecutar el addon, o por el contrario no permitir que sea ejecutado.

    Whitelist & Blacklist

    Primero, debes tener la UUID de la(s) persona(s) que utilizará el addon protegido, o que estará en la blacklist. Escribiendo msx_uuid en el menú principal imprimirá el UUID actual.

    Debes tener en línea un archivo de extensión .json con el siguiente formato:
    Whitelist: Habilitado solo para algunos 

    1

    "allowed": [
        "uuid1",

    1

    {
    "denied": [
        "uuid1",
        "uuid2",
        "etc."
    ]
}

    Ahora, en la sección main de tu addon, utilizarás el siguiente comando con esta sintaxis. 

    1
    a_protected <link al archivo .json>

    Codificación

    Para codificar tus addons, debes tener activado el modo debug. En el menú principal, escribirás crypt, luego ingresarás el nombre de tu .addon o de tu .py (Debe estar en la carpeta de addons).
    El addon codificado quedará en la carpeta de addons, con el nombre de nombre_encoded.extensión; puedes compartirlo y funcionará como cualquier otro addon.
